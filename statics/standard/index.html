'基本规范'全部展开
    规范目的

      为提高团队协作效率, 便于后台人员添加功能及前端后期优化维护,前端开发人员必须按本文档规范进行前端页面的开发； 本文档如有不对或者不合理的地方请及时提出, 经讨论决定后方可更改。

    基本准则
    • 符合web标准；
    • 结构/表现/行为 分离；
    • 语义化html；
    • 代码书写简洁明了有序；
    • 测试需兼容Firefox,Chrome,Opera,IE(ie8及以上版本)浏览器;
    • 尽可能减小服务器负载, 保证最快的解析速度。
    文件命名原则
    • 命名优先考虑对应英文，可使用常规缩写形式，如英文太生僻或复杂再考虑拼音
    • 命名尽可能使得你自己和工作组的每一个成员能够理解每一个文件的意义；
    • 同一类的文件使用一样的前缀，使得同类文件都能排列在一起，如： shopping //商城首页 shopping-cart //购物车 shopping-order //订购清单
    图片规范
    案例图片
    • 案例图片指前端页面用img元素调用的图片，一般为后台可配置的图片
    • 存放路径：项目根目录的/images文件夹内
    • 在保证视觉效果的情况下选择最小的图片格式与图片质量, 以减少加载时间
    • 非首页轮播大图外的图片大小尽量限定在 70K 左右或更小
    • 图片格式仅限于gif || png || jpg
    背景图片
    • 背景图片指样式中使用background属性设置的图片，一般为比较少改动的背景图或小图标
    • 存放路径：项目根目录的/css/imgs文件夹内
    • 项目发布前背景图需分类合并，以减小http请求
    • 图片格式仅限于gif || png || jpg
    • 透明背景的图标用png24格式；非透明背景的图标如图形扁平色彩单一的用png8格式，否则用jpg格式

    html基本结构
  1. 使用tab键缩进代码
  2. 推荐使用html5的文档声明<!DOCTYPE HTML>，且在文档的第一行，位于html标签之前
  3. 必须申明文档的编码charset，且与文件本身编码保持一致，推荐使用UTF-8编码 <meta charset="utf-8">
  4. 模板参看：html基本结构
    html标签
    html标签使用
  1. h1 - 用于网页主题，一个页面只有一个
  2. h2 - 用于大栏目标题 or 楼层标题 等
  3. h3 - 用于小栏目标题
  4. h4 - 用于列表每一条目的标题
  5. h5,h6 - 用于除以上情况的其他如强调加粗等情况
  6. ul+li - 列表最佳元素组合
  7. p - 用于描述简介等文本
  8. span - 用于搭配其他元素用以布局标签等，尽量不要用于外容器
  9. table - 表格型数据
    10. html标签格式
  10. 标签要闭合，不能省略可选的结束标签，如：<div>的结束标签</div>不能省略
    11. html标签优化
  11. 页面元信息的<title>元素不可缺少，控制在25个字、50个字节以内。“二级栏目 - 一级栏目 - 网站名称”。
  12. 表单的radio控件和checkbox控件的标签文本使用label元素包裹，或使用label的for属性确保点击标签文本时能激活对应控件，如： <label>check-1:<input type="checkbox" /></label> 或<lable for="check2">check-2:</lable><input type="checkbox" id="check2" />
    html属性
    html属性顺序
    请尽量按以下顺序排列标签属性：
  1. class
  2. id,name
  3. data-*
  4. src,for,type,href
  5. title,alt
  6. aria-*
    7. 实例如下： <a class="..." id="..." data-modal="toggle" href="#"> Example link </a> <input class="form-control" name="control" type="text" /> <img src="..." alt="..." />
    html属性格式
  7. html标签属性使用双引号包裹，不使用单引号
  8. ID属性在一个页面内是唯一的
  9. 外调文件如果是默认类型时统一不用声明type，如： <link rel="stylesheet" href="xxx.css" /> <script src="xxx.js"></script>
  10. 一般情况下不使用内置style
  11. 合理设置链接的target属性，如：target='_blank'
    12. html属性优化
  12. a标签如无链接时其属性href值填充':javascript:;'
  13. img标签需要添加alt属性以增加可访问性<img src="" title="图片描述" alt="图片描述">
  14. a链接如锚文本不足以描述此链接，需要加title属性，如： <a title="力德网站" href="#">点击这里</a>
  15. 元信息的keywords关键词发布前请添加合适内容，针对SEO。（关键词控制在5个左右,单个8汉字以内）
  16. 元信息的description网站描述发布前请添加合适内容，针对SEO。（字数尽量控制在80个汉字，160个字符以内）
  17. 布尔型属性在声明时统一不赋值，如： <input type="text" disabled> <input type="checkbox" value="1" checked> <select> <option value="1" selected>1</option> </select>
  18. 对于属性值或颜色参数，省略小于 1 的小数前面的 0 ，如： .5 代替 0.5 -.5px 代替 -0.5px
    html文本&注释
  1. 带有实体名称的 ASCII 实体或特殊字符要使用实体名（实体名称对大小写敏感）；
    请参看：html实体名对照表
  2. 网页每个模块都应该注释其栏目名称或简短说明，注释格式如下（允许只有开始注释）： <!--头部--> ... <!--/头部-->

    css基本设置
    重置样式
  1. 模板参看：reset.css参考设置
    2. 页面编码
  2. 统一使用 UTF-8 编码,用@charset "utf-8"指定页面编码
  3. 全局字体设置： windows 7系统：微软雅黑 Arial；<br /> windows XP及以下：新宋体，宋体，Arial<br /> MAC默认字体：Helvetica Neue和Helvetica Hiragino Sans GB。<br /> font-family:"Microsoft Yahei","Hiragino Sans GB","Helvetica Neue",Helvetica,tahoma,arial,Verdana,sans-serif,"WenQuanYi Micro Hei","\5B8B\4F53";
  4. 中文字体使用编码转换，参考： 微软雅黑 \5FAE\8F6F\96C5\9ED1 或 Microsoft YaHei<br /> 黑体 \9ED1\4F53<br /> 新宋体 \65b0\5b8b\4f53<br /> 宋体 \5b8b\4f53
  5. 推荐使用颜色参考：216web安全色
    css属性
    css属性声明顺序
  1. 模板参看：css属性声明顺序
    2. css属性格式
  2. 当属性值需要加引号时，请使用双引号，如： font-family:"Microsoft Yahei",arial,Tahoma,sans-serif; content:"xxx"; input[type="text"]
  3. 每条声明独占一行，如： a:hover, a.on{...}
  4. 每个样式属性后的分号";"不能省略
  5. 属性值尽量使用缩写，如： border:0 10px; margin:10px 15px 20px; padding:10px; color:#efg; background:#eee url(xxx.png) no-repeat center right;
    6. css属性优化
  6. 样式都用class设置，慎用id
  7. 基于减少类名设置、防止属性设置冲突、保持结构清晰的目的，尽量使用css选择器，如： ul > li{...} input[type=text]{...}
  8. 选择器尽量减少不必要的嵌套层级，如： 一个外容器类名为p-list的模块内只有一个ul和ul下几个同级的li，设置li样式时用.p-list .li{...}而不用.p-list ul li{...}
  9. 相同或类似的模块尽量把代码整合重用起来
  10. 针对特殊浏览器的属性，应写在标准属性之前，例如：-webkit-box-shadow:;-moz-box-shadow:;box-shaow:;
  11. 带有前缀的属性，单独一行，通过缩进，让每个属性的值垂直对齐，方便编辑维护
  12. 避免使用各种CSS Hack，如需对 IE 进行特殊定义，请参阅：CSS Hack说明
  13. .hover，selected，disabled，current等具有特定意义的请勿直接占用。
  14. .:link :visited :hover :active书写顺序 L-V-H-A，速记：LoVe（喜欢）HAte（讨厌）。
    css属性值
    css属性值格式
  1. 值为0时后面不加单位
  2. 颜色值英文的都用小写
    3. css属性值优化
  3. css慎用expression表达式
  4. 慎用 !important
  5. 避免使用兼容性不好的滤镜
  6. 上下模块之间的间距统一使用下一个模块的margin-top来实现，好处是：如果没有下一个模块也不会多出一段空隙
    css导入&注释
  1. 不要使用 @import导入样式表
  2. 注释格式，统一使用双斜杠加*。
  3. 需注释所属栏目或模块标题，跟html对应模块标题一致; 多个模块标题用'&'分隔，格式如下： /*案件列表 & 新闻列表*/ .p-list-pics{...}

'js规范'全部展开
    js基本设置
  1. 传统js脚本的编写，要求使用下面的模板进行包裹： (function($){ 'use strict'; // code... }(jQuery));
  2. requirejs脚本的编写，要求使用下面的格式： define([ "jquery", "angular" ], function ($, angular) { 'use strict'; // code... });
    js格式
  1. 每句语句后的';'不能省略
  2. if、while、for、do语句的执行体总是用"{"和"}"括起来，即使在其结构体内只有一条语句
  3. 代码编写时应该优先使用单引号(‘)，防止字符串可能包含带有双引号(“)的HTML代码
  4. 总是使用var声明变量，避免全局变量的污染
    js优化
  1. 使用闭包时防止循环引用
  2. 尽量避免使用存在兼容性及消耗资源的方法或属性, 比如eval() & innerText
  3. 避免对象上使用with()
  4. 避免数组使用for in
    js注释
  1. 脚本模块需写明所属页面或栏目，注释必要时尽量详尽且到位
  2. 插件等独立脚本文件需添加详细的头描述，格式参考如下： /* * 简述 * * 功能详细描述 * * @param <String> arg1 参数1 * @param <Number> arg2 参数2，默认为0 * @return <Boolean> 看xxx是否成功 */ +(function(){ 'use strict'; var pluginName = "xxx"; ... })();
